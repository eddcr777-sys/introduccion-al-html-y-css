Que es HTML?

HTML es un lenguaje de marcado que utiliza una serie de etiquetas para definir distintos tipos de contenido en una página web. Cada etiqueta tiene un propósito específico, como definir encabezados, párrafos, listas, enlaces, e imágenes. Los navegadores web interpretan estas etiquetas para mostrar el contenido de manera visual.

Su Historia

La evolución de HTML ha sido fundamental para el desarrollo de la web. Su historia se puede resumir en las siguientes etapas clave: 1. Creación (1989-1991) Tim Berners-Lee, un físico del CERN, inventó HTML en 1989 como parte de la creación de la World Wide Web. La primera versión se conocía como “HTML 1.0” y contenía un conjunto limitado de etiquetas. 2. HTML 2.0 (1995) Se formalizó en 1995 como HTML 2.0, el primer estándar oficial, que incluyó más etiquetas y mejoras de funcionalidades. 3. HTML 3.2 (1997) Introducidos nuevos elementos como tablas y applets. Adoptado por el World Wide Web Consortium (W3C) que se encargaba de estandarizar el lenguaje. 4. HTML 4.01 (1999) Se publicó en diciembre de 1999, consolidando muchos de los elementos de versiones anteriores y permitiendo un mejor soporte multimedia. Se separaron las preocupaciones de contenido y presentación, promoviendo el uso de CSS. 5. XHTML 1.0 (2000) Se introdujo XHTML como una reformulación de HTML 4.01 usando la sintaxis de XML, lo que obligó a una mayor rigurosidad en el marcado. 6. HTML5 (2014) HTML5 fue desarrollado para abordar las limitaciones de versiones anteriores y mejorar la compatibilidad con dispositivos móviles. Incluyó soporte para multimedia (audio y video), gráficos, y nuevas APIs. HTML5 se convirtió en un estándar en 2014 y continúa evolucionando con mejoras y nuevas features.

1. Estructura de un Documento HTML

Estructura Básica de un Documento HTML Un documento HTML típico sigue una estructura básica que incluye los siguientes elementos:

Elemento Descripción
<!DOCTYPE html> Declara el tipo de documento y la versión de HTML.
<html> Elemento raíz que contiene todo el contenido del documento.
<head> Contiene metadatos, enlaces a estilos, scripts, etc.
<title> Define el título de la página.
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Asegura que el diseño sea responsivo, ajustando el ancho a la del dispositivo.
<meta charset="UTF-8"> Define la codificación de caracteres del documento como UTF-8, que admite una amplia gama de caracteres.
<link rel="stylesheet" href="estilos.css"> Vincula un archivo CSS externo para dar estilo al documento.
<body> Contiene todo el contenido visual de la página que se presentará a los usuarios.

2. Etiquetas HTML

Las etiquetas HTML se utilizan para estructurar el contenido. A continuación, se enumeran algunas etiquetas comunes junto con su significado semántico:

Etiqueta Descripción
<h1> - <h6> Encabezados, donde <h1> es el más importante y <h6> el menos.
<p> Define un párrafo.
<a>Crea un enlace.
<img>Inserta una imagen.
<ul>Lista desordenada.
<ol>Lista ordenada.
<li>Elemento de una lista.
<div>Contenedor genérico para agrupar contenido.
<span>Contenedor en línea, generalmente para estilizar.
<header>Encabezado de una sección o de todo el documento.
<footer>Pie de página de una sección o de todo el documento.
<article>Contenido independiente en un documento.
<section>Sección temática dentro de un documento.
<aside>Contenido relacionado pero separado del contenido principal.
<form>Formulario para recopilación de datos.
<input>Campo de entrada en un formulario.
<button>Botón interactivo.
<table>Crea una tabla.
<tr>Fila en una tabla.
<td>Celda de una tabla.
<th>Encabezado de una celda en una tabla.
<blockquote>Cita en bloque.
<nav>Define un conjunto de enlaces de navegación.
<main>Especifica el contenido principal de un documento.
<figure>Agrupa contenido multimedia, como imágenes, con una leyenda (figcaption).
<figcaption>Define una leyenda para un elemento <figure>.
<details>Crea un widget de divulgación en el que la información solo es visible cuando se activa.
<summary>Define un encabezado visible para un elemento <details>.
<time>Define una fecha u hora específica.
<mark>Resalta parte del texto.
<progress>Muestra el progreso de una tarea.
<meter>Representa una medida escalar dentro de un rango conocido.
<label>Define una etiqueta para un elemento <input>.
<textarea>Define un control de entrada de texto de varias líneas.
<select>Crea una lista desplegable.
<option>Define una opción en una lista desplegable.
<optgroup>Agrupa opciones relacionadas en una lista desplegable.
<fieldset>Agrupa elementos relacionados en un formulario.
<legend>Define un título para un elemento <fieldset>.
<audio>Inserta contenido de audio.
<video>Inserta contenido de video.
<source>Define múltiples recursos de medios para <video> y <audio>.
<track>Especifica pistas de texto para <video> y <audio>.
<canvas>Se utiliza para dibujar gráficos a través de scripts (generalmente JavaScript).
<svg>Define un contenedor para gráficos vectoriales SVG.
<iframe>Incrusta otro documento HTML en la página actual.
<b>Pone el texto en negrita sin importancia semántica.
<i>Pone el texto en cursiva sin importancia semántica.
<u>Subraya el texto sin importancia semántica.
<strong>Indica que el texto tiene una gran importancia (se muestra en negrita).
<em>Enfatiza el texto (se muestra en cursiva).
<code>Define un fragmento de código de computadora.

3.Semántica en HTML

Semántica en HTML La semántica en HTML se refiere al uso de etiquetas que tienen significados claros y específicos sobre el tipo de contenido que representan. Al utilizar etiquetas semánticas, se mejora la accesibilidad, el SEO (optimización para motores de búsqueda) y la mantenibilidad del código. A continuación, se detalla la importancia y las principales etiquetas semánticas en HTML.

Etiqueta Significado
<header> Encabezado de una sección o de todo el documento.
<nav> Menú de navegación.
<article> Contenido independiente en un documento.
<section> Sección temática dentro de un documento.
<footer> Pie de página de una sección o de todo el documento.
<main> Contenido principal de una página.
<aside> Contenido relacionado, pero separado del contenido principal.
<figure> Representación gráfica o ilustrativa con caption.
<figcaption> Título o descripción de la figura.

4. Atributos y Valores

Los atributos proporcionan información adicional sobre los elementos HTML y siempre se especifican en la etiqueta de apertura. Algunos de los atributos más comunes son:

Atributo Descripción
idIdentificador único para un elemento.
classDefine una o más clases para aplicar estilos.
styleAplica estilos CSS en línea.
srcFuente de un recurso, como una imagen o un script.
hrefURL de destino para un enlace.
altTexto alternativo para imágenes.
titleTexto que aparece como un tooltip.
valueValor de un campo de entrada.
placeholderTexto mostrado en un campo de entrada cuando está vacío.
disabledIndica que un campo de entrada está deshabilitado.
targetEspecifica dónde abrir el documento enlazado (ej. "_blank").
readonlyEspecifica que un campo de entrada es de solo lectura.
requiredIndica que un campo de formulario debe ser completado antes de enviarlo.
checkedEspecifica que un elemento de entrada debe estar preseleccionado.
selectedEspecifica que una opción en una lista desplegable debe estar preseleccionada.
typeDefine el tipo de un elemento, como un <input> o un <button>.
forAsocia una etiqueta <label> con un elemento de formulario.
actionEspecifica la URL a la que se enviarán los datos de un formulario.
methodDefine el método HTTP para enviar datos de un formulario (ej. "GET", "POST").
langEspecifica el idioma del contenido de un elemento.

Ejemplos de Atributos de Formulario: `action` y `method`

Los atributos action y method son cruciales para definir cómo y dónde se envían los datos de un formulario.

Ejemplo con method="GET"

El método GET anexa los datos del formulario a la URL especificada en el atributo action. Es útil para formularios de búsqueda o cuando los datos no son sensibles, ya que los datos son visibles en la barra de direcciones del navegador. La longitud de la URL es limitada.

<form action="/buscar" method="get">
    <label for="busqueda">Buscar:</label>
    <input type="text" id="busqueda" name="q">
    <button type="submit">Enviar</button>
</form>

Si un usuario escribe "html" en el campo de búsqueda y envía el formulario, el navegador navegará a una URL como esta: /buscar?q=html.

Ejemplo con method="POST"

El método POST envía los datos del formulario en el cuerpo de la solicitud HTTP. Es el método preferido para enviar datos sensibles (como contraseñas) o grandes cantidades de información, ya que los datos no son visibles en la URL.

<form action="/registrar" method="post">
    <label for="usuario">Usuario:</label>
    <input type="text" id="usuario" name="usuario">
    <label for="clave">Contraseña:</label>
    <input type="password" id="clave" name="clave">
    <button type="submit">Registrarse</button>
</form>

Al enviar este formulario, los datos usuario y clave se envían de forma oculta al script /registrar en el servidor para ser procesados.
